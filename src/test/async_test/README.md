# async_test — WebSocket broadcast сервис

Тестовый сервис на [userver](https://userver.tech/), реализующий WebSocket чат с глобальным broadcast-каналом.

## Как работает

При подключении нового клиента к `/ws` создаётся запись в общем хранилище (`NotifierClient`). Для каждого соединения запускаются два параллельных корутинных цикла:

- **ReadLoop** — читает входящие сообщения и рассылает их всем подключённым клиентам (`broadcast`)
- **SendLoop** — ждёт сообщений в очереди (через condition variable) и отправляет их этому клиенту

Любое сообщение от любого клиента мгновенно получают все участники (включая отправителя). Один глобальный канал, без комнат и аутентификации.

## Эндпоинт

| Метод | Путь | Описание |
|-------|------|----------|
| GET   | `/ws` | WebSocket соединение (порт 8080) |

## Сборка и запуск

**Первая сборка + запуск:**
```bash
make new
```
Выполняет `cmake -B .build && cmake --build .build`, затем запускает сервис.

**Запуск уже собранного бинаря:**
```bash
make run
```

Сервис поднимается на `http://localhost:8080`. Логи пишутся в `log.log` (файл очищается при каждом запуске).
